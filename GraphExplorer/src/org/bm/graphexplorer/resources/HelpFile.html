Help for GraphExplorer

This file contains limited help for GraphExplorer. An alternate source for help is the project wiki at http://blaisegraphexplorer.googlecode.com/.

GraphExplorer is still under active development, and you may have some ideas about additional things you would like it to do, or you may experience some bugs. Whether it's an additional capability or the software breaks, we'd appreciate feedback. Please visit http://blaisegraphexplorer.googlecode.com/ to submit feedback, or email the project owner at triathematician@gmail.com, subject line "Graph Explorer".

Getting Started

There are two ways to get started: either generate a new graph, or load an existing graph. Once a graph is loaded, GraphExplorer can automatically layout the graph, or compute various metrics on the graph.

Generating New Graphs

GraphExplorer can generate several kinds of fixed and random graphs. Currently supported are empty, circle, star, wheel, and complete graphs. Each of these may be generated with arbitrary numbers of nodes, as directed or undirected graphs.

Currently, five types of random graphs may be generated. Most of these have additional parameters that may be adjusted:

  • Uniform (Erdos-Renyi) random graph: Allows either a fixed number of edges or a fixed edge probability
  • Degree sequence random graph: Edges are randomly generated while ensuring a fixed degree sequence in the resulting graph
  • Watts-Strogatz random graph: Graph is obtained starting with a lattice graph after rewiring edges with a fixed probability
  • Preferential attachment graph: Graph is obtained from a starting graph after adding additional nodes using the preferential attachment algorithm. Allows adding either a fixed number of edges per step, or a randomly determined number. The resulting graph may be either static or longitudinal, in which case a slider will be generated to show how the graph is constructed over time.
  • Proximity random graph: nodes are randomly placed within a box, and connected if they are within a certain distance of each other. This feature is experimental, and additional parameters will be added in later versions.

Loading Existing Graphs

GraphExplorer supports several standard graph file types (although it does not support every option within these types). This allows the user to load existing graphs, or to edit a text file to describe the graph. Currently supported file types include:

  • Edge list format: each line consists of a pair of integers indicating the edge between two nodes.
  • Pajek format (.net): vertex declarations (with positions), and a variety of input formats for edges, including edge lists and adjacency matrices.
  • Extended pajek format (.net,.netx,.netl): pajek format with additional custom file information
  • UCINet format (.dat): supports more general data structures
  • GraphML format (.graphml): some limited support currently
The Pajek format is the default for GraphExplorer, and has the most widely supported set of options. In the future, we hope to add support for additional formats.

Once a file is loaded, the File Editor under the View menu shows the text content of the file. A capability currently planned for future versions is the ability to edit the graph directly from that view.

For more information about these file types, including brief descriptions, see the wiki page at http://blaisegraphexplorer.googlecode.com/.

Graph Layout

GraphExplorer contains a number of static layout options, as well as the ability to animate an iterative layout as it is being applied. The static layouts are all accessible as buttons from the toolbar:

  • Circular Layout: Places nodes around a circle.
  • Random Layout: Places nodes at random within a box.
  • Static Spring Layout: Uses simulated physics to generate ideal locations for nodes and edges. This layout gives the best results, but may take some time to converge for large graphs. The algorithm is described further below.

The spring layout may also be animated to let the user watch the algorithm working over time. At each step, the algorithm determines a number of forces on the nodes, and moves the nodes appropriately. The forces are:

  1. a repulsive force between each pair of vertices
  2. an attractive force between each pair of connected vertices
  3. a global force ensuring components stay close to the middle of the screen
Parameters for this layout are shown in the property panel on the lefthand side of the window.

There is an additional Time Spring Layout option for longitudinal graphs, accessible under the Layout menu. This generates a static layout for the slices in a dynamic graph. This is an experimental feature and has not been fully tested.

Visualization and Appearance Features

Basic visual settings may be adjusted using the property panel at left. These settings include background color, node size/shape/color, edge color/thickness, and label style. There is also a checkbox to turn labels on and off (under the "Labels" box).

Still images of the current view may be saved in a variety of common formats, and .mov files of longitudinal graphs may also be exported.

Metrics

Currently, one metric at a time may be displayed along with the graph. The metric displays in several ways:

  • The node sizes on the main plot are adjusted to have areas proportional to that node's metric value.
  • The actual metric values are displayed on the main graph table.
  • A chart & table of the distribution of metric values is shown at bottom.
We do not currently have concise descriptions of these metrics available.

Longitudinal Graphs

This application supports longitudinal graphs, which change over time. A sample longitudinal graph may be created using the "preferential attachment" random graph generator, and selecting the longitudinal option. Special features included with the longitudinal view include:

  1. A slider below the main graph that displays the time frame for the longitudinal graph, which may be adjusted dynamically. If a dynamic layout is active, the layout automatically reconfigures for the new graph.
  2. A chart displaying the value of the selected metric over time. For this to be active, one or more nodes must be highlighted: use the Layout -> Highlight Nodes to select these nodes.
